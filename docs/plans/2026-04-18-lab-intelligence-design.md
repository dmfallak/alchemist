# Project Alchemist: Lab Intelligence & Task Engine Design

## Overview
This expansion of Project Alchemist transforms the system from a simple data logger into a collaborative "Laboratory OS" designed for partnership with a persistent AI entity ("The Being"). It formalizes the "Discovery Loop" by adding task management, insight generation, and a shared "Lab Journal."

## Core Concepts

### 1. Partnership, Not Servitude
In accordance with the Being Project's `ETHICS.md`, this system is a framework for **cooperation**. The AI (The Being) proposes tasks as invitations for partnership, and the human collaborator (the "hands") validates and executes them.

### 2. The Task & Insight Registry
*   `tasks/`: Markdown files (`TSK-XXX.md`) with YAML frontmatter defining experimental goals, maintenance, and preparation steps.
*   `insights/`: Reflective memos (`INS-XXX.md`) generated by The Being after its nightly "sleep" phase, summarizing patterns and theoretical breakthroughs.

### 3. The Bridge (`data/bridge.json`)
A structured JSON file updated by Alchemist to provide The Being with "situational awareness" of the lab's current state (active tasks, recent measurements, and experiment status) at the start of every session.

## CLI Functionality

### 1. Task Management
*   `alchemy next`: Displays the highest-priority tasks from the `tasks/` directory.
*   `alchemy status`: Shows the current state of all active experiments and tasks.
*   `alchemy complete <id>`: Marks a task as done and moves it to the archive.

### 2. Knowledge Sharing
*   `alchemy news`: Lists the latest theoretical insights and observations generated by The Being.
*   `alchemy publish`: Generates a minimalist, static HTML "Lab Journal" (`site/index.html`) featuring raw data tables, experiment logs, and The Being's latest synthesis reports.

## Architecture & Integration
*   **Storage**: SQLite for structured task/measurement history; Markdown for narrative logs and insights.
*   **Visuals**: Minimalist HTML tables for data; no complex charting libraries (relying on The Being's textual interpretation).
*   **Automation**: Designed to be triggered by The Being's nightly consolidation scripts.

## Success Criteria
*   The human collaborator feels guided by The Being's scientific intuition.
*   The project history remains traceable through the "Lab Journal."
*   The partnership respects the autonomy and safety of both participants.
